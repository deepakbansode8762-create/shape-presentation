üéØAbstract Classes

An abstract class cannot be instantiated but provides a blueprint for subclasses.

  • ‚úÖ Can contain both abstract and concrete methods
  • ‚úÖ Forces subclasses to implement abstract methods
  • ‚úÖ Enables code reusability and polymorphism
  • ‚úÖ Provides a common interface for related classes
abstract class Shape { // Abstract methods - must be implemented by subclasses public abstract double calculateArea(); public abstract double calculatePerimeter(); }

class Circle extends Shape { private double radius; // Constructor public Circle(double radius) { this.radius = radius; } @Override public double calculateArea() { return Math.PI * radius * radius; } @Override public double calculatePerimeter() { return 2 * Math.PI * radius; } }

üî∫Triangle Class Implementation

class Triangle extends Shape { private double side1, side2, side3; // Constructor public Triangle(double side1, double side2, double side3) { this.side1 = side1; this.side2 = side2; this.side3 = side3; } @Override public double calculateArea() { double s = (side1 + side2 + side3) / 2; // Semi-perimeter return Math.sqrt(s * (s - side1) * (s - side2) * (s - side3)); // Heron's formula } @Override public double calculatePerimeter() { return side1 + side2 + side3; } }

üíªComplete Main Class

public class Shape_peri_area_Main { public static void main(String[] args) { // Create Circle object with radius 5.0 Circle circle = new Circle(5.0); // Create Triangle object with sides 3.0, 4.0, 5.0 Triangle triangle = new Triangle(3.0, 4.0, 5.0); // Display Circle information System.out.println("Circle:"); System.out.println("Area: " + circle.calculateArea()); System.out.println("Perimeter: " + circle.calculatePerimeter()); System.out.println(); // Display Triangle information System.out.println("Triangle:"); System.out.println("Area: " + triangle.calculateArea()); System.out.println("Perimeter: " + triangle.calculatePerimeter()); } }

üîëKey OOP Concepts

  • üì¶ Abstraction: Abstract class defines structure without implementation
  • üîÑ Inheritance: Circle and Triangle extend the Shape class
  • üé≠ Polymorphism: Same method names with different implementations
  • üîí Encapsulation: Private fields with public methods
  • ‚úçÔ∏è Method Overriding: @Override annotation ensures correct implementation
  • üèóÔ∏è Code Reusability: Common interface reduces code duplication

üöÄExtending the Program

You can easily add more shapes by extending the Shape class:

// Adding a Rectangle class class Rectangle extends Shape { private double length, width; public Rectangle(double length, double width) { this.length = length; this.width = width; } @Override public double calculateArea() { return length * width; } @Override public double calculatePerimeter() { return 2 * (length + width); } }

The abstract class design makes it easy to add new shapes without modifying existing code!
